# 架构说明

## 系统概述

Harness Agent System 是一个基于 LangGraph 和 MCP（模型上下文协议）的智能 Agent 平台，面向生产级 AI Agent 部署设计。

## 核心组件

### 1. API 层 (`src/app/api/`)

- **FastAPI** REST 端点，用于 Agent 交互
- 支持同步 (`/invoke`, `/stream`) 和异步 (`/ainvoke`, `/astream`) 操作
- 服务器发送事件 (SSE) 流式响应

### 2. Agent 层 (`src/app/agents/`)

- **LangGraph** Agent，支持可配置的中间件链
- 可插拔的检查点和存储，用于状态持久化
- 人机交互支持，用于关键操作审批

### 3. 工具 (`src/app/tools/`)

- **内置工具**: 天气查询、PDF 解析、文件操作
- **MCP 工具**: 通过模型上下文协议可扩展
- 通过 `mcp_client_builder` 动态注册工具

### 4. 中间件 (`src/app/middleware/`)

- **PDFParseMiddleware**: PDF 附件自动处理
- **HumanInTheLoopMiddleware**: 审批工作流
- **ToolCallLimitMiddleware**: 使用频率控制
- **ModelFallbackMiddleware**: 模型优雅降级

### 5. 处理器 (`src/app/processors/`)

- **PyMuPDF4LLM**: 快速 PDF 文本提取
- **Docling**: 支持多模态的高级 PDF 解析
- **PDF Cache**: 基于 SHA-256 的解析结果缓存

### 6. MCP 服务器 (`src/app/mcp_server/`)

- **FecMall MCP**: 电商平台集成 (70+ 工具)
- 通过 MCP 协议可扩展的工具生态

## 数据流

```
用户请求
    │
    ▼
FastAPI 处理器
    │
    ▼
Agent.invoke() / agent.astream()
    │
    ├──► 中间件链
    │       │
    │       ├──► PDF 中间件 (解析附件)
    │       ├──► 模型降级 (主模型失败时回退)
    │       ├──► 人机交互 (需要审批时中断)
    │       └──► 工具调用限制 (频率控制)
    │
    ├──► LLM 调用 (DeepSeek / 豆包)
    │       │
    │       └──► 工具执行 (如需要)
    │               │
    │               ├──► 内置工具
    │               └──► MCP 工具 (FecMall 等)
    │
    └──► 状态更新 → 检查点 (SQLite)
    │
    ▼
响应 (JSON / SSE 流)
```

## 配置管理

所有配置通过环境变量和 `.env` 文件管理，使用 `pydantic-settings` 进行类型安全的加载。

| 组件 | 配置前缀 | 说明 |
|------|----------|------|
| AI 模型 | `DEEPSEEK_*`, `VISION_*`, `OPENAI_*` | LLM 提供商设置 |
| PDF 解析器 | `PDF_*` | PDF 解析行为 |
| 安全 | `API_KEY`, `CORS_*`, `RATE_LIMIT_*` | 认证与安全 |
| 服务器 | `SERVER_*` | 主机、端口、工作进程数 |
| 日志 | `LOG_LEVEL` | 日志详细程度 |
