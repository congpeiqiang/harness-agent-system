# 贡献指南

感谢你对 Harness Agent System 的关注！以下是参与贡献的方式。

## 开发环境搭建

1. Fork 并克隆本仓库
2. 安装 uv 包管理器
3. 运行 `uv sync` 安装依赖
4. 复制 `.env.example` 为 `.env` 并配置
5. 运行 `make dev` 启动开发服务器

## 代码规范

- 使用 Ruff 进行代码格式化和 Lint
- 使用 MyPy 进行类型检查
- 使用 Conventional Commits 规范编写提交信息

```bash
make format    # 格式化代码
make lint      # 检查代码质量
make typecheck # 类型检查
```

## 测试

```bash
make test      # 运行所有测试
make test-unit # 仅运行单元测试
```

## 提交规范

- `feat`: 新功能
- `fix`: 修复 Bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建/工具变更

## Pull Request 流程

1. 从 `main` 分支创建特性分支
2. 进行修改并添加测试
3. 确保所有检查通过
4. 提交 PR 并填写描述
5. 等待 Code Review

## 问题反馈

- 使用 GitHub Issues 报告 Bug
- 使用 GitHub Discussions 进行功能讨论

## 许可证

贡献的代码将在 MIT 许可证下发布。
